## 核心特性

### 1. 独立上下文
- 每个 Agent 有自己的 MazeState
- 不能直接访问其他 Agent 的内存
- 只能通过 State Channel 通信

### 2. 关卡系统
- 每个单元格有一个关卡
- 关卡类型：
  - 🔢 数学题（需要计算 skill）
  - 🔍 逻辑题（需要推理 skill）
  - 🗝️ 密码锁（需要解密 skill）
  - 🧩 谜题（需要模式识别 skill）
  - 🤝 协作题（需要其他 Agent 帮助）

### 3. Skill 系统
- 每个 Agent 有独特的 skill 池
- Skill 可以共享（通过 Hub）
- 某些关卡需要特定 skill

### 4. 通信协议
- StateMessage: 发送当前状态
- HelpRequest: 请求帮助
- SkillShare: 分享 skill
- CoordinationMessage: 协调行动

## 迷宫配置

| Agent | 迷宫大小 | 关卡数 | 特殊技能 |
|-------|---------|--------|---------|
| Agent 1 | 4x4 | 16 | 数学计算 |
| Agent 2 | 8x8 | 64 | 逻辑推理 |
| Agent 3 | 12x12 | 144 | 密码解密 |
| Agent 4 | 16x16 | 256 | 模式识别 |

## 通信示例

```
Agent 1 → Hub: "卡在 (2,3)，需要逻辑推理 skill"
Hub → Agent 2: "Agent 1 需要逻辑推理帮助，位置 (2,3)"
Agent 2 → Hub: "发送逻辑推理 skill 给 Agent 1"
Hub → Agent 1: "收到来自 Agent 2 的逻辑推理 skill"
Agent 1 → Hub: "关卡解决，继续前进"
```

## 运行方式

```bash
# 启动协作系统
python3 collaborative/main.py

# 指定 Agent 数量
python3 collaborative/main.py --agents 4

# 详细日志
python3 collaborative/main.py --verbose
```
